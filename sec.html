Understanding Cybersecurity & Security Concepts

Malware & Attacks

Malware Overview

Malware refers to any software intentionally designed to disrupt, damage, or gain unauthorized access to computer systems. It includes viruses, worms, trojans, spyware, adware, keyloggers, backdoors, and botnets.

Examples of Malware Types

  • Viruses: Programs that attach to files and spread when files are shared. Example: ILOVEYOU virus.
  • Worms: Self-replicating malware that spreads across networks. Example: Conficker.
  • Spyware: Secretly collects user info. Example: keyloggers.
  • Adware: Displays unwanted ads. Example: Fireball adware.
  • Bots: Remote-controlled programs forming botnets. Example: Zeus botnet.
  • Trojans: Malware disguised as legitimate software. Example: Emotet.
  • Keyloggers: Record keystrokes to steal passwords.
  • Backdoors: Hidden ways to access systems without authorization.
  • DoS Attacks: Overwhelm systems to make them unavailable. Example: Ping of Death.

Other attacks include Tmining, where system resources are exploited, and network-based attacks like Ping of Death. Spoofing and sniffers are techniques used to intercept or falsify network communications.

Security Principles & Methods

Importance of CIA

The CIA triad represents Confidentiality, Integrity, and Availability. Confidentiality ensures data is accessible only to authorized users. Integrity ensures data is accurate and unaltered. Availability ensures that systems and information are accessible when needed. Together, they form the foundation of cybersecurity.

Safety Methods

AAA (Authentication, Authorization, Accounting) are fundamental safety methods. Authentication verifies user identity, Authorization grants access based on permissions, and Accounting logs user activities to monitor and prevent misuse.

Security Layers & Key Terms

Security Layers

Security is implemented in layers including physical security, network security, host security, application security, and data security. Each layer reduces risk and provides multiple barriers against attackers.

Key Cybersecurity Terms

  • Asset: Anything valuable that needs protection, such as data or systems.
  • Vulnerability: Weakness in a system that can be exploited.
  • Threat: Potential cause of harm to an asset.
  • Threat Agent: Entity responsible for exploiting a vulnerability.
  • Risk: Likelihood of a threat exploiting a vulnerability.
  • Attack Vectors: Paths or methods used to breach security.
  • Control: Measures taken to prevent or mitigate threats.

Network Security

How Network Security is Achieved

Network security is enforced through Firewalls, IDS (Intrusion Detection Systems), and VPNs. Firewalls filter traffic to block unauthorized access. IDS monitors network activity and alerts for suspicious behavior. VPNs encrypt connections, providing secure communication over untrusted networks. Together, these tools form a strong network defense strategy.

Digital Forensics

Stages of Forensics

  • Identification: Recognizing potential digital evidence.
  • Preservation: Ensuring the evidence is not altered or destroyed.
  • Analysis: Examining evidence to extract meaningful information.
  • Documentation: Recording findings in detail for reports.
  • Presentation: Presenting evidence clearly, often in court or internal reports.
